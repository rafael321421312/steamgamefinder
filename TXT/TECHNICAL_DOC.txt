⚙️3. Техническая документация
1. Структуры данных в коде
* Словари (dict): Используются для хранения тем (`self.themes`), кэша цен (`self.price_cache`), и данных об игре, возвращаемых API.
* Списки (list): Хранение результатов поиска (`self.current_results`), списка избранных игр (`self.favorites`).
* Множества (set): Используются для уникальных жанров (`self.available_genres`, `self.selected_genres`) для быстрого поиска и исключения дубликатов.
* Потоки (Thread): Фоновые потоки `threading.Thread` используются для методов `perform_real_search` и `load_game_data`, чтобы интерфейс Tkinter не "зависал" во время сетевых запросов.

2. Алгоритмы
А. Алгоритм поиска с fallback (резервированием)
1.Попытка 1: Запрос к `search/suggest` (быстрый автокомплит).
2.Попытка 2 (Fallback): Если результат пуст, парсинг HTML страницы `store.steampowered.com/search/` с помощью регулярного выражения `r'href=".../app/(\d+)/..."'`.
3.Попытка 3 (Локальный fallback): Поиск по захардкоженному списку популярных игр (Portal, Dota 2 и т.д.).

Б. Алгоритм кэширования (TTL - Time To Live)
При запросе цены (`get_real_price`):
1. Проверяется наличие `appid` в `self.price_cache`.
2. Если есть, вычисляется разница: `time.time() - timestamp`.
3. Если разница `< 3600` (1 час), данные возвращаются из кэша.
4. Если `< 86400` (24 часа) при загрузке общего кэша, данные не удаляются. Иначе кэш очищается.

В. Алгоритм очистки HTML (Парсинг требований)
Метод `clean_requirements` использует цепочку регулярных выражений для удаления HTML-тегов (`<[^>]+>`), замены спецсимволов (`&nbsp;`, `&lt;`) и схлопывания лишних пробелов (`\s+`), превращая сырой HTML в читаемый текст.

3. Сетевое взаимодействие (IP и Протоколы)
* Протокол: HTTPS (порт 443).
* Целевые домены (IP-адреса динамические, за CDN Steam):
    *`store.steampowered.com` (API и веб-поиск).
    *`steamdb.info` (открывается в браузере пользователя).
* Заголовки: Для обхода базовых Cloudflare/бот-защит используется подмена `User-Agent`: `'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'`.
* IP-адрес клиента: Приложение использует публичный IP-адрес пользователя для обращения к Steam. Steam не требует API-ключа для базовых методов `appdetails`, поэтому авторизация не требуется.

4. Описание "Базы данных" (Локальное хранилище)
Важное примечание: В текущей реализации НЕ используется реляционная СУБД (MySQL/PostgreSQL). Данные хранятся в локальных файлах (NoSQL / Flat-file). Ниже описана их структура.

А. `settings.json` (Настройки)
json {
"theme": 
"dark"
}


Б. `favorites.json` (Избранное)
Массив объектов.
json
[
  {
    "name": "Portal 2",
    "appid": "620",
    "url": "https://store.steampowered.com/app/620/",
    "price": "199.00 RUB (-50%)",
    "added_date": "2026-06-03 12:00:00",
    "note": ""
  }
]

В. `steam_cache.pkl` (Кэш цен)
Бинарный файл Python (Pickle). Внутри представляет собой словарь:
`{ "appid": {"current_price": float, "original_price": float, "discount": int, "currency": str, "timestamp": float} }`

(Если бы использовалась **MySQL**, схема БД выглядела бы так:)*
Таблица `games` (id INT PK, appid VARCHAR UNIQUE, name TEXT, url TEXT).*
Таблица `prices` (id INT PK, game_id INT FK, current_price DECIMAL, discount INT, updated_at TIMESTAMP).*
Таблица `favorites` (id INT PK, game_id INT FK, added_date DATETIME).*

5. Инструкция по установке и запуску
Требования
* Операционная система: Windows 10/11 (или Linux/macOS).
* Python версии 3.8 или выше.
* Подключение к интернету.

Шаг 1: Установка Python
Скачайте и установите Python с официального сайта [python.org](https://www.python.org/). При установке **обязательно** отметьте галочку *"Add Python to PATH"*.
Шаг 2: Установка зависимостей
Откройте командную строку (Win+R -> `cmd`) и выполните команду для установки единственной внешней библиотеки:
`bash
pip install requests
```
*(Библиотеки `tkinter`, `json`, `pickle` и др. уже встроены в стандартную библиотеку Python).*

Шаг 3: Запуск приложения
1. Сохраните код в файл с именем `steam_finder.py`.
2. Запустите его двойным кликом или через командную строку:
```bash
python steam_finder.py
```

Шаг 4: Использование
При первом запуске в папке со скриптом автоматически создадутся файлы `settings.json`, `favorites.json` и `steam_cache.pkl`. Не удаляйте их, если хотите сохранить свои настройки и избранное.
